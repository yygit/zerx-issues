Рефакторинг граббера

Dmitrii Prisacari created an issue 2016-05-11

В процессе рефакторинга вызова функций curl (#598) столкнулись с тем, что мы не можем нормально отладить работу граббера по нескольким причинам: во-первых код, вежливо говоря, "с душком", во-вторых, нет четкого понимания, как он должен работать.

Поэтому, после завершения задачи со страницей редактирования #443, нужно переписать модуль граббера. После того, как он будет переписан, заканчиваем задачу в рефакторингом curl (#598).

Далее изложу информацию по грабберу, которую собрал во время анализа:

  1. Страница со списком фильмов и скачиваемых файлов

    Здесь есть возможность добавить сериал на скачивание. Запись, добавленная через форму на скриншоте, попадает в таблицу seasonvar.

  2. После того, как отправляется форма создания новой записи, производится переход на страницу с сериями.

    Здесь выбирается список серий, которые нужно добавить на скачивание. Только когда выбрана хотябы одна серия, идет вставка записи в таблицу seasonvar. Производится проверка размера всех выбранных серий, и затем они фиксируются в таблице seasonvar_files. Последнюю таблицу обновляет внешний сервер по мере скачивания файла.

Серии сервер получает в результате:

  • Первого запроса на страницу с плеером для получения ссылки к списку серий.

  • Второго запроса по ссылке со списком серий. Здесь я заметил, что в ссылке для каждой серии в хеше-ключе (он разный для каждого IP) не достает одного символа. Возможно это сделано специально:

Особое внимание следует уделить разделению логики, простому и понятному именованию переменных и функций.

При рефакторинге граббера нужно учесть запрос из задачи #190, если это возможно.
